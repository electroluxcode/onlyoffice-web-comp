# 公文审阅页对接说明

事件分两层：**宿主页面用 EventBus**，**编辑器内 Word 行为用 `manager.subscribe`**。

---

## 第一层：`onlyofficeEventbus`

组件库在打开/保存等流程里往总线抛事件，React 页直接监听或 `waitFor`。

```ts
import { onlyofficeEventbus } from '@/components/onlyoffice-comp/lib/eventbus'
import { ONLYOFFICE_EVENT_KEYS } from '@/components/onlyoffice-comp/lib/const'
```

| 事件 | 何时用 |
|------|--------|
| `DOCUMENT_READY` | 文档打开完成 → 再调修订/批注 API |
| `SAVE_DOCUMENT` | `export()` 完成，带 `binData`（一般交给 `exportEditorDocumentAsFile`，少直接监听） |
| `LOADING_CHANGE` | 显示/隐藏 Loading |
| `ONSAVE` | 保存流程结束，无 `binData` |


```ts
onlyofficeEventbus.on(ONLYOFFICE_EVENT_KEYS.ONSAVE, () => { /* ... */ })

```

多实例时 `SAVE_DOCUMENT` 用 `data.instanceId === manager.getInstanceId()` 过滤。

---

## 第二层：`manager.subscribe`

iframe 里 OnlyOffice Word SDK 的 `asc_on*` 回调，订修订、批注、脏状态等细粒度变化。

```ts
const manager = editorManagerFactory.create(containerId)

// 须在 DOCUMENT_READY 之后
await onlyofficeEventbus.waitFor(ONLYOFFICE_EVENT_KEYS.DOCUMENT_READY, 30000)

const off = await manager.subscribe({
  type: 'asc_onDocumentModifiedChanged',
  fn: (modified) => setHasChanges(!!modified),
})

// 卸载
off()
```
